# Claude Code 源码深度调研报告 (更新版)

本报告基于对 `Claude Code` (最新版 v4.6) 源码的深度审计，详细分析了其在 **Claude 4.6 (Opus & Sonnet)** 时代的核心架构演进、模型感知调度以及极致的工程化实践。

---

## 1. 核心架构与推理循环 (The Brain)

`Claude Code` 的核心是一个高度稳健的推理引擎 (`QueryEngine.ts`)，它在 4.6 版本中引入了更精细的模型感知逻辑。

### 1.1 自我修复与容错机制
*   **最大输出 Token 自动恢复**：
    *   **设计**：当生成超长代码被截断时，系统自动触发恢复循环（最多 3 次），利用上下文关联请求模型“继续执行”。
    *   **价值**：解决了处理大型重构（如 1000+ 行文件）时的中断痛点。
*   **抢占式会话记录 (Pre-emptive Transcription)**：
    *   **设计**：在发起 API 请求**之前**，立即将用户消息写入磁盘。
    *   **价值**：确保即使在 API 响应前进程被杀，会话依然可完美恢复。

### 1.2 上下文压缩与微型摘要 (Micro-compaction)
*   **智能历史裁剪 (History Snip)**：通过 `snipReplay` 精准“剪除”冗余过程，保留核心决策。
*   **工具结果预算 (Tool Result Budgeting)**：自动摘要超长的工具输出（如海量 `grep` 结果），防止撑爆上下文。

---

## 2. Claude 4.6 专项优化 (Model-Aware Strategy)

源码显示该项目已深度集成 **Opus 4.6** 和 **Sonnet 4.6**，并针对其特性设计了差异化策略。

### 2.1 动态推理努力度 (Dynamic Effort Steering)
*   **Opus 4.6 独有：Max Effort (`EFFORT_MAX`)**：
    *   源码中明确 `max` 级别的推理努力度仅限 Opus 4.6。非 4.6 模型会被自动降级为 `high`。
    *   **实现**：系统根据任务复杂度（通过 `advisor.ts` 判断）动态建议或自动调整 `effort` 参数。
*   **默认倾向**：Pro 用户在使用 Opus 4.6 时，系统默认推荐 `medium` 努力度以平衡速度与深度。

### 2.2 百万级上下文管理 (1M Context Support)
*   **Sonnet 4.6 (1M)**：源码中包含大量 `coral_reef_sonnet` 实验代码，专门处理 100万 Token 级别的上下文缓存与计费逻辑。
*   **Opus 1M Merge**：系统会实时监测上下文占用，并在触发阈值时弹出 `Opus 1M` 合并通知。

## 3. 工具调度与执行 (The Hands & Feet)

### 3.1 智能工具分区调度 (Concurrency Partitioning)
*   **逻辑**：系统自动识别工具调用的副作用。
    *   **并行执行**：连续的“只读”工具（如 `FileRead`, `Glob`, `LSPTool`）并行执行（并发度 10）。
    *   **串行执行**：涉及副作用的工具（如 `FileWrite`, `BashTool`）降级为串行，确保环境一致性。

### 3.2 深度 LSP 集成
*   通过 `LSPTool` 直接对接语言服务器，提供比正则匹配更精准的代码重构建议。

---

## 4. 交互体验设计 (The Heart)

### 4.1 Ultraplan：混合架构模式
*   设计：通过 `/ultraplan` 命令将复杂任务“传送”到云端 Web 界面。用户在 Web 端微调计划，批准后 CLI 自动同步结果。

### 4.2 Buddy 系统：情感化反馈
*   设计：基于 UserID 种子生成的唯一“终端伴侣”。通过 `Ink` 实时展示 AI 的任务进度和“心跳”，大幅缓解用户等待焦虑。
